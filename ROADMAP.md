# ROADMAP — ระบบอำนวยการยิงปืนใหญ่ (Artillery FCS) สู่การใช้งานระดับประเทศ

> เอกสารนี้ออกแบบมาเพื่อ "ป้อน Opus ทีละขั้น" — แต่ละหัวข้อ `STEP` คือ 1 prompt ที่สั่งทำได้จบในตัว
> โครงสร้างไฟล์ปัจจุบัน: `fdc/` (เว็บ+คำนวณ), `fdc_dongle/` (เฟิร์มแวร์แม่ข่าย), `gun_unit/` (เฟิร์มแวร์หมู่ปืน)

---

## 0. สรุประบบปัจจุบัน (ของจริงที่มีแล้ว)

| ส่วน | ฮาร์ดแวร์ | ไฟล์ | หน้าที่ |
|------|----------|------|--------|
| หมู่ปืน | LILYGO T-Beam S3 + RM3100 | `gun_unit/gun_unit.ino` | GPS + เข็มทิศ → ส่งพิกัด/มุมภาค LoRa, รับคำสั่งยิง, นำกระบอกปืนเข้าทิศ |
| เครื่องแม่ข่าย | Raspberry Pi + T-Beam (dongle) | `fdc_dongle/fdc_dongle.ino` | สะพาน LoRa ↔ USB Serial (JSON 115200) |
| เว็บ ศอย. | รันบน Pi/PC | `fdc/main.py`, `fdc/fdc_radio.py`, `fdc/ballistic.py`, `fdc/templates/index.html` | แผนที่, คำนวณหลักฐานยิง, สั่งยิง, realtime |

**โปรโตคอล LoRa ปัจจุบัน** (868 MHz, SF12, BW62.5, CR5, sync 0xAB):
- `0x20` GUN_STATUS (หมู่ปืน→แม่ข่าย): lat,lon,az,el,bat,hdg
- `0x10` FIRE_CMD (แม่ข่าย→หมู่ปืน): gun,seq,az,el,charge
- `0x11` ACK, `0x01` PING, `0x02` PONG
- ป้องกันข้อมูลด้วย CRC16 เท่านั้น — **ยังไม่มีการเข้ารหัส/ยืนยันตัวตน**

**ช่องว่างที่ต้องแก้ก่อนใช้งานจริงระดับประเทศ** (สรุปไว้ก่อน รายละเอียดอยู่ในแต่ละก้อน):
1. ตรวจจับเครื่องแม่ข่ายช้า/ไม่ชัวร์ เวลาเสียบเครื่องอื่น (โจทย์หลักของรอบนี้)
2. LoRa ไม่เข้ารหัส — ความเสี่ยงด้านความมั่นคง
3. ความถี่ 868 MHz เป็นย่าน EU — ไทยต้องใช้ **AS923 (920–925 MHz)** ตามกฎ กสทช.
4. addressing เล็กเกินไป (หมู่ปืน 1–4, แม่ข่ายตัวเดียว) — ระดับประเทศต้องมี node ID/หน่วยเป็นลำดับชั้น
5. ตารางยิงเป็นค่าจำลอง — ต้องใส่ตารางยิงจริงตามชนิดปืน
6. ยังไม่มีชั้นเชื่อมขึ้นบน (ผบ. ที่อื่นเห็นภาพรวม) และยังไม่มีฐานข้อมูล/สิทธิ์ผู้ใช้

## 1. สถาปัตยกรรมเป้าหมาย (4 ชั้น)

```
ชั้น 3  ศูนย์บัญชาการระดับประเทศ (National COP)
        Server กลาง + ฐานข้อมูล + เว็บ ผบ. เห็นทุกหน่วยทั้งประเทศ
            ▲  (Internet/4G/SATCOM/วิทยุข้อมูล — MQTT over TLS)
ชั้น 2  ศูนย์บัญชาการระดับกองพัน/พื้นที่ (Regional)
        รวมหลาย ศอย. เห็นเฉพาะหน่วยในสังกัด
            ▲
ชั้น 1  ศอย. (FDC node)  ← "เครื่องแม่ข่าย" + เว็บ ทำงานได้เองแม้เน็ตหลุด
            ▲  (LoRa AS923)
ชั้น 0  หมู่ปืน (Gun units)  ← T-Beam ที่กล้องเล็งปืน
```

**หลักการออกแบบสำคัญ:**
- **Edge autonomy**: ศอย.แต่ละจุดทำงานครบวงจรได้เองออฟไลน์ (รับพิกัดหมู่ปืน, คำนวณ, สั่งยิง) ไม่พึ่งเน็ต
- **Store-and-forward**: เก็บข้อมูลไว้ในเครื่อง ส่งขึ้นบนเมื่อมีลิงก์ ไม่หายเมื่อเน็ตหลุด
- **Transport-agnostic**: ชั้นบนเชื่อมด้วยอะไรก็ได้ (4G/Starlink/วิทยุข้อมูลทหาร) — ใช้ MQTT เป็นกลาง

- **ความปลอดภัยเป็นค่าเริ่มต้น**: เข้ารหัสตั้งแต่ LoRa ถึง uplink, ยืนยันตัวตนทุก node

## ก้อนงาน A — Plug-and-Play เครื่องแม่ข่าย (โจทย์หลักรอบนี้)
**เป้าหมาย:** เสียบ dongle เข้าคอมเครื่องไหนก็ได้ (PC/โน้ตบุ๊ก, Windows/Linux/Mac) แล้ว **ตรวจเจอ + เชื่อมต่อภายใน ~1 วินาที** อย่างชัวร์ ไม่จับผิดอุปกรณ์

### STEP A1 — เพิ่ม handshake ยืนยันตัวตนในเฟิร์มแวร์แม่ข่าย
ไฟล์: `fdc_dongle/fdc_dongle.ino`
- ใน `processCmd()` เพิ่มคำสั่ง `WHOAMI` → ตอบกลับทันที:
  `{"type":"IDENT","role":"FDC_DONGLE","fw":"v2","proto":1,"node":1}`
- ตอน boot ให้พ่นบรรทัด IDENT เดียวกันซ้ำ 3 ครั้งห่างกัน 200ms (กันพลาดตอนเพิ่งเสียบ)
- เหตุผล: ฝั่ง PC จะ "ถาม-ตอบ" เพื่อยืนยันว่าใช่ dongle เราจริง ไม่ใช่เดาจาก VID เฉยๆ

### STEP A2 — เขียน auto-detect ใหม่แบบ active probe + ขนาน
ไฟล์: `fdc/fdc_radio.py` (แก้ `_find_port`)
- รวบรวมพอร์ตผู้สมัครจาก VID/PID ให้ครบ: ESP32-S3 CDC `0x303A`, CP210x `0x10C4`, CH340/CH9102 `0x1A86`, FTDI `0x0403`
- เปิดทุกพอร์ตผู้สมัคร **พร้อมกัน (thread)** ส่ง `{"type":"WHOAMI"}` รอ `IDENT` ภายใน 800ms
- พอร์ตไหนตอบ IDENT ถูก = ตัวจริง เลือกอันนั้น (รองรับเสียบหลาย dongle: เก็บ map node→port)
- ลด `RECONNECT_DELAY` 3.0 → 1.0 วิ
- เหตุผล: เร็ว + ไม่จับผิด แม้มี Arduino/อุปกรณ์ serial อื่นเสียบอยู่

### STEP A3 — ตรวจจับการเสียบ/ถอดแบบ hotplug (ไม่ต้องรอ poll)
ไฟล์: `fdc/fdc_radio.py`
- Linux: ใช้ `pyudev` monitor subsystem `tty` → เสียบปุ๊บ connect ปั๊บ
- Windows/Mac: fallback เป็น poll `comports()` ทุก 1 วิ เทียบ diff
- ส่ง event ขึ้นเว็บ (`radio_connected` / `radio_disconnected`) ให้ UI แสดงสถานะทันที

### STEP A4 — แพ็กเกจรันง่ายทุกเครื่อง (one-click)
- ทำ `fdc/requirements.txt` + สคริปต์ `run.sh` / `run.bat`
- PyInstaller สร้างไฟล์เดียว: `fdc-server.exe` (Windows), binary (Linux/Mac) — ดับเบิลคลิกแล้ว **เปิดเบราว์เซอร์ไปที่ `localhost:5000` อัตโนมัติ**
- เพิ่มในโค้ด: `webbrowser.open()` ตอนสตาร์ท
- Linux: แถมไฟล์ udev rule `99-fdc.rules` (ให้สิทธิ์ /dev/tty โดยไม่ต้อง sudo) + เอกสารติดตั้งไดรเวอร์ CP210x สำหรับ Windows
- แนะนำ: ใช้ **USB CDC ในตัว ESP32-S3** (ไม่ต้องลงไดรเวอร์บน Windows 10/11) เป็นช่องหลัก

### STEP A5 — แสดงสถานะการตรวจจับบนเว็บ
ไฟล์: `fdc/templates/index.html`, `fdc/main.py`
- แถบบนสุด: "RADIO: CONNECTED (COM5 / node 1)" หรือ "SEARCHING…"
- ปุ่ม "Rescan" เรียก auto-detect ใหม่

## ก้อนงาน B — ความปลอดภัย + ความถูกต้องตามกฎหมายความถี่
**ทำก่อนขยายระดับประเทศ — เป็นข้อบังคับ ไม่ใช่ทางเลือก**

### STEP B1 — ย้ายความถี่เป็น AS923 (ไทย)
ไฟล์ทั้งสอง `.ino`
- `LORA_FREQ 868.0` → ย่าน **920–925 MHz** (เช่น 920.6/922.0/923.4) ตามแผนช่องสัญญาณ กสทช.
- ทำเป็น `#define` ช่อง + ตารางช่องไว้ใช้วางแผนความถี่หลายหน่วย (กันชนกันระหว่างกองร้อยข้างเคียง)

### STEP B2 — เข้ารหัส + ยืนยันความถูกต้องของแพ็กเก็ต LoRa
ไฟล์ทั้งสอง `.ino`
- ใส่ **AES-128-CCM** (เข้ารหัส + MAC) แทน CRC16 เปล่า — มี key ต่อกองร้อย
- ใส่ **nonce/sequence + anti-replay** กันการบันทึกแล้วส่งซ้ำ (สำคัญมากกับคำสั่งยิง)
- เก็บ key ใน NVS (`Preferences`) ตั้งค่าผ่านโหมด provisioning

### STEP B3 — ขยาย addressing เป็นลำดับชั้น
- เปลี่ยน gun ID 1–4 → โครงสร้าง `unitId(หน่วย) + gunId` รองรับหลายกองร้อย/กองพัน
- เพิ่มฟิลด์ node/unit ในแพ็กเก็ตและใน state ฝั่งเว็บ

### STEP B4 — ใส่ตารางยิงจริง
ไฟล์: `fdc/ballistic.py`
- แทน `FIRING_TABLE` จำลอง ด้วยตารางยิงจริงตามชนิดปืนที่หน่วยใช้ (เลือกชนิดปืนได้)
- เพิ่มการแก้ความสูง (alt diff), อุณหภูมิ/ลม (ถ้ามีข้อมูลตรวจอากาศ), ขนาดกระสุน

## ก้อนงาน C — ชั้นเชื่อมขึ้นบน (ศอย. → ศูนย์บัญชาการ)
**เป้าหมาย:** ส่ง telemetry/สถานะของทุก ศอย. ขึ้นไปรวมที่ส่วนกลาง โดยทำงานได้แม้ลิงก์ไม่เสถียร

### STEP C1 — เพิ่ม MQTT client ใน ศอย.
ไฟล์ใหม่: `fdc/uplink.py`
- ศอย.เป็น MQTT client เผยแพร่ไปยัง broker ส่วนกลาง:
  - `fdc/<unit>/<node>/fdc_gps`, `.../gun/<id>/status`, `.../fire_mission`, `.../event`
- รับคำสั่งจากเบื้องบน (เช่น ภารกิจยิงที่สั่งจากศูนย์) ผ่าน topic `cmd/...`
- ใช้ QoS 1 + **offline queue** (เก็บลง SQLite ในเครื่อง ส่งซ้ำเมื่อกลับมาออนไลน์) → store-and-forward
- TLS + client certificate ต่อ node (mTLS)
- เชื่อมเข้ากับ state เดิม: hook ที่ `radio_poller()`/`push_sse()` ให้ publish ขึ้น MQTT ด้วย

### STEP C2 — ตั้งค่า node identity
ไฟล์: `fdc/config.yaml`
- ระบุ unitId, nodeId, ชื่อหน่วย, พิกัดฐาน, endpoint broker, path ใบรับรอง
- โหลดตอนสตาร์ท แสดงบนเว็บ

## ก้อนงาน D — Server ส่วนกลาง + Dashboard ผู้บังคับบัญชา
**เป้าหมาย:** ผบ. นั่งที่ไหนก็เปิดเว็บเห็นทั้งประเทศ — มีอะไรอยู่จุดไหน กำลังจะทำอะไร

### STEP D1 — โครงสร้างโปรเจกต์ server กลาง
โฟลเดอร์ใหม่: `central/`
- **MQTT broker** (Mosquitto/EMQX) รับ telemetry จากทุก ศอย.
- **Ingest service** (Python/FastAPI): subscribe MQTT → เขียนฐานข้อมูล → push realtime
- **ฐานข้อมูล**: PostgreSQL + **PostGIS** (เชิงพื้นที่) + **TimescaleDB** (time-series telemetry)
- **Realtime**: WebSocket ส่งอัปเดตเข้าเบราว์เซอร์ ผบ.

### STEP D2 — เว็บ Dashboard ระดับประเทศ
โฟลเดอร์: `central/web/`
- แผนที่ประเทศ: หมุดทุกหน่วย (ศอย./หมู่ปืน/เป้าหมาย), zoom เข้าดูรายหน่วย
- Panel รายหน่วย: สถานะหมู่ปืน, แบต, สัญญาณ, ภารกิจยิงที่กำลังทำ
- ไทม์ไลน์/ประวัติเหตุการณ์ + replay ย้อนหลัง
- Live "fire mission in progress" feed

### STEP D3 — สิทธิ์ผู้ใช้ + ความปลอดภัย
- Auth (OIDC/JWT), **RBAC**: ผบ.ระดับประเทศเห็นทุกหน่วย, กองพันเห็นเฉพาะสังกัด, ศอย.เห็นของตน
- Audit log ทุกคำสั่ง, เข้ารหัส at-rest/in-transit
- แยกสิทธิ์ "ดู" กับ "สั่งยิง" ชัดเจน

### STEP D4 — ประวัติ/รายงาน
- บันทึกทุกภารกิจยิง, สถิติ, export รายงานหลังปฏิบัติ (AAR)

## 3. สิ่งที่ต้องตัดสินใจ (รอข้อมูลจากผู้ใช้)

- **ความถี่/ใบอนุญาต**: ช่อง AS923 ที่ได้รับอนุญาตให้หน่วยใช้ (มีผลต่อ B1)
- **สื่อ uplink ชั้นบน**: 4G/Starlink/วิทยุข้อมูลทหาร — กำหนดดีไซน์ C1
- **ชนิดปืน + ตารางยิงจริง**: เพื่อใส่ใน B4
- **จำนวนหน่วยสูงสุด**: กองร้อย/กองพัน/หมู่ปืน — กำหนดแผน addressing (B3) และความจุ server (D1)
- **โฮสต์ server กลาง**: on-prem ในเครือข่ายทหาร หรือ cloud
- **การจัดการกุญแจเข้ารหัส**: ใครออก/หมุนเวียน key (B2) และใบรับรอง (C1)
```
